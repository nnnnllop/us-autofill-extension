# AutoFill

Расширение для Chrome: автозаполнение billing-адреса и тестовых карт на страницах оплаты.

Поддерживаются **Stripe**, **Shopify**, **Paddle**, **Lemon Squeezy** и обычные HTML-формы.

## Возможности

- **Адрес и имя** — 10 стран (US, GB, DE, FR, CA, AU, NL, IT, ES, PL), публичные адреса + имена через [randomuser.me](https://randomuser.me)
- **Генерация карт** — алгоритм Луна, настраиваемый BIN, пресеты MC / Visa / Amex
- **Проверка карт** — всегда запускается через публичные API (namso.live, chkr.cc) после создания новой карты
- **Профили** — до 50 наборов (страна, BIN, email), экспорт/импорт JSON
- **Закрепление адреса** — улица и город не меняются, обновляется только имя
- **Stripe Checkout** — кнопка **Заполнить** на странице, пошаговый retry, заполнение iframe
- **Редкие поля карты** — `Cardholder name`, `Country or region` и `ZIP` в payment-блоке заполняются из текущего профиля/адреса
- **Отчёт заполнения** — какие поля заполнены (✓), пропущены (·), в iframe (↗)
- **Горячая клавиша** — `Ctrl+Shift+Z` (настраивается в Chrome)
- **Shadow DOM** — поиск полей внутри shadow root
- **Автозаполнение** — по умолчанию выключено; при включении показывается предупреждение, что функция ещё дорабатывается
- **Dev-логи** — можно включить, скачать, скопировать и очистить прямо в popup

## Установка

### Из исходников (режим разработчика)

1. Скачайте или клонируйте репозиторий:
   ```bash
   git clone https://github.com/nnnnllop/us-autofill-extension.git
   ```
2. Откройте `chrome://extensions`
3. Включите **Режим разработчика**
4. Нажмите **Загрузить распакованное расширение**
5. Выберите папку `us-autofill-extension`

### Обновление иконок (опционально)

```bash
cd icons
pip install pillow
python generate_icons.py
```

## Быстрый старт

1. Откройте popup расширения (иконка в панели Chrome)
2. Сохраните свой **email** — на сайте он не перезапишется, если уже заполнен
3. Нажмите **Обновить всё** — сгенерируются адрес и карта
4. На странице оплаты:
   - нажмите **Заполнить → Всё** в popup, или
   - на Stripe — кнопка **Заполнить** в углу, или
   - `Ctrl+Shift+Z`

Автоматическое заполнение при загрузке страницы выключено по умолчанию. Его можно включить в настройках popup, но это beta-функция: она ещё дорабатывается, возможны баги на отдельных сайтах.

В версии 2.3 карта создаётся сразу и автоматически уходит на проверку. Пока API отвечает, в popup отображается статус проверки; если API недоступен или ограничивает запросы, карта остаётся Лун-валидной и готовой для тестового заполнения.

## Popup — основные элементы

| Элемент | Описание |
|---------|----------|
| **Профиль** | Переключение наборов данных. Двойной клик — переименовать |
| **Email** | Сохраняется между сессиями и в профиле |
| **📌 Закрепить** | Фиксирует адрес (улица, город, индекс) |
| **Экспорт / Импорт** | Бэкап профилей в JSON |
| **Заполнить** | Всё / Адрес / Карта. Перед ручным заполнением расширение пытается доинжектить content script в текущую вкладку, поэтому кнопка работает и на уже открытых вкладках после перезагрузки расширения |
| **Копирование** | Клик по полю в панели — в буфер обмена |

### Настройки (футер popup)

- **Автозаполнение** — при загрузке страницы; по умолчанию выключено, при включении показывается предупреждение о beta-статусе
- **Режим авто** — всё / адрес / карта
- **Звук** — сигнал при успешном заполнении
- **Компактный** — скрыть превью карты
- **Кнопка на сайте** — плавающая кнопка **Заполнить** на checkout.stripe.com; можно скрыть
- **Сводка** — строка `Профиль · MC · US · email`
- **Dev-логи** — включить сбор, скачать `.txt`, скопировать в буфер или очистить
- **Настроить** — изменить горячую клавишу (`chrome://extensions/shortcuts`)

## Приватность и данные

| Где хранится | Что |
|--------------|-----|
| `chrome.storage.local` (ваш браузер) | Профили, email, BIN, закреплённый адрес, кэш карты, настройки, dev-логи при включении |
| Код расширения | Только публичные адреса-заготовки и дефолтный BIN |

**Не коммитьте** экспортированные файлы `autofill-profiles*.json` — в них могут быть ваши email и BIN.

### Внешние запросы

- `randomuser.me` — генерация имён
- `namso.live`, `chkr.cc` — проверка созданных тестовых карт

Расширение не отправляет ваши сохранённые профили на сторонние серверы.

## Changelog

### 2.4.2

- Реализовано принудительное перезаполнение полей: при ручном клике на кнопку «Заполнить» в popup или на плавающую кнопку «Заполнить» (FAB) на Stripe-странице, расширение теперь перезаписывает все поля, даже если они уже заполнены. При автоматическом автозаполнении (на загрузке страницы) пустые поля заполняются без перезаписи введенных пользователем данных.

### 2.4.1

- Ограничено количество попыток поиска карт до 5 за сессию для предотвращения блокировок со стороны API.
- Добавлена задержка в 2 секунды между попытками проверки карт для более плавного расхода лимитов API.
- Данные о здоровье API (cooldown) теперь сохраняются в `chrome.storage.local` и сохраняются при перезапуске Manifest V3 Service Worker.
- Улучшен парсинг ответов API для надежного разделения прикладных серверных лимитов и ошибок.

### 2.4.0

- Обязательная параллельная двойная проверка карт по обоим API (`namso.live` и `chkr.cc`).
- Карта помечается как `API: APPROVED` только при подтверждении обоими API (Вариант Б).
- Добавлена гибкая компромиссная логика: если `chkr.cc` вернул ошибку лимита/таймаута, но `namso.live` одобрил карту, она одобряется со статусом `APPROVED (LIMIT)` с предупреждением в UI (проверка не прерывается).
- Если хотя бы один из API отклонил карту (`DECLINED`), она бракуется и генерируется новая.
- Убрано ограничение на количество попыток проверки карт, поиск продолжается в рамках общего таймаута.
- Общий таймаут проверки увеличен до 3 минут (`180000 мс`).
- В popup встроен новый UI-блок с подробными деталями проверки по каждому API (источник, статус, код, сообщение, время ответа в мс).

### 2.3.1

- Выбор валюты ограничен реальными currency-контролами: кнопки оплаты и другие действия больше не рассматриваются как варианты валюты.
- Исправлен fallback между `namso.live` и `chkr.cc` при таймаутах, rate limit и недоступности API.
- Проверяются ровно две созданные карты; третья непроверенная карта больше не подменяет результат.
- Состояние проверки сохраняется и восстанавливается через `chrome.alarms` после остановки Manifest V3 service worker.
- Автозаполнение запускается на Stripe и повторяется для динамически появившихся форм.
- Ручное заполнение получило повторные попытки, добавлена корректная поддержка `textarea`.

### 2.3.0

- Карты создаются сразу и всегда запускают внешнюю проверку.
- Удалён выключатель API-проверки, чтобы поведение было единым и предсказуемым.
- Popup показывает текущий статус проверки карты и продолжает работать при таймаутах/rate limit.
- Улучшена готовность к публичной версии: понятнее поведение настроек и стабильнее ручное заполнение.
